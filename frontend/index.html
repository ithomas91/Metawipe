metawipe

GPS, kameraadatok, IPTC, XMP és ICC profil eltávolítása képekből. A feldolgozás a böngésződben fut — a fájl nem kerül fel semmilyen szerverre.

# húzd ide a képet, vagy

JPEG, PNG · több fájl is kijelölhető · max. 30 MB / fájl

Ugyanaz a tisztítás, REST API-n keresztül — automatizáláshoz, batch feldolgozáshoz, szerver oldali integrációhoz.

Más bizalmi szint, tudatosan. A fenti Tisztító fül soha nem hívja ezt az API-t — minden alapból a böngésződben marad. Ha ezt az API-t hívod, a fájl elhagyja az eszközödet és felmegy a szerverre. A feldolgozás memóriában történik, semmi nem kerül lemezre vagy logba, de ez egy explicit, más döntés, mint a kliens-oldali eszköz használata.
GET/api/v1/health
Életjel-ellenőrzés. Visszaadja a futó API verzióját.
POST/api/v1/inspect
Visszaadja, milyen metaadatot talált a feltöltött képben — törlés nélkül.
filemultipart/form-data, kötelező — a kép (JPEG vagy PNG)
válaszJSON: camera, gps, has_icc_profile, has_xmp, iptc, has_thumbnail, …
POST/api/v1/strip
Egy fájl megtisztítása. Alapból minden metaadatot töröl.
filemultipart/form-data, kötelező
keep_copyright"true" / "false" (alapérték: false) — megtartja a Copyright mezőt, ha van
keep_artist"true" / "false" (alapérték: false) — megtartja a Készítő/Artist mezőt, ha van
válasza megtisztított fájl bináris tartalma (image/jpeg vagy image/png)
POST/api/v1/strip/batch
Több fájl egyszerre. Ugyanazok az opciók, mint a /strip végpontnál.
filesmultipart/form-data, több fájl, ugyanazon a mezőnéven (files)
válaszZIP, fájlonként a megtisztított verzióval
curl -X POST /api/v1/strip \
  -F "file=@photo.jpg" \
  -F "keep_copyright=true" \
  -o photo_clean.jpg
import requests

with open("photo.jpg", "rb") as f:
    r = requests.post(
        "/api/v1/strip",
        files={"file": f},
        data={"keep_copyright": "true"},
    )

with open("photo_clean.jpg", "wb") as out:
    out.write(r.content)
const form = new FormData();
form.append("file", fileInput.files[0]);
